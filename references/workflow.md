## 资源管理示例

### 写作前添加人物

在开始写作前，可以先添加主要人物：

```bash
# 添加主角
/char add 林远 "主角，少年，坚毅，林府灭门唯一幸存者" "清风（救命恩人）、林父（父亲）"

# 添加重要配角
/char add 清风 "散修，路过救下林远，身份神秘" "林远（救命恩人）"

# 查看人物列表
/char list
```

**输出：**
```
## 人物列表（《仙途》）

| 人物名 | 概况 | 相关资源 |
|--------|------|----------|
| 林远 | 主角，少年，坚毅，林府灭门唯一幸存者 | 清风、林父 |
| 清风 | 散修，路过救下林远，身份神秘 | 林远 |

共 2 个人物
```

### 添加设定

```bash
# 添加宗门设定
/set add 青云宗 "主角所在宗门，正道大派，历史悠久" "修仙体系、林远（弟子）"

# 添加力量体系设定
/set add 修仙体系 "练气→筑基→金丹→元婴→化神→渡劫→飞升" "青云宗、林远"

# 查看设定列表
/set list
```


## 冲突检测详细规则

### 人物冲突检测
1. 检查人物身份是否与已有设定矛盾
2. 检查人物关系是否与人物库一致
3. 检查人物行为是否符合性格设定（反差/伪装除外）
4. 检查已死亡人物是否再次出现
5. 检查人物名称是否重复
6. **检查 list.md 中的人物关系是否对等**

### 时间线冲突检测
1. 检查事件顺序是否合理
2. 检查时间间隔是否符合逻辑
3. 检查人物年龄/修为是否与时间线匹配
4. 检查章节编号是否连续

### 设定冲突检测
1. 检查新设定是否与总设定库冲突
2. 检查力量体系是否一致
3. 检查地理/势力设定是否矛盾
4. 检查设定名称是否重复
5. **检查 list.md 中的相关资源是否存在**

### 伏笔冲突检测
1. 检查已回收伏笔是否再次触发
2. 检查伏笔 ID 是否重复
3. 检查伏笔揭露方式是否与计划一致
4. 检查伏笔状态是否正确更新

### 情节冲突检测
1. 检查本章情节是否与总纲冲突
2. 检查本章情节是否与卷纲冲突
3. 检查本章情节是否与已写章节矛盾
4. 检查卷纲与实际章节数是否匹配

## Git 操作说明

### 自动提交
每次完成章节写作后自动执行：
```bash
git add 库/
git commit -m "完成《{书名}》第 X 卷第 N 章：[章节标题]"
```

### 手动提交
用户可随时执行 `/git commit` 手动提交。

### 远程同步
需用户提前配置远程仓库，支持自动 `git push`。

### Commit 信息规范
- 完成章节：`完成《书名》第 X 卷第 N 章：[标题]`
- 修改章节：`修改《书名》第 X 卷第 N 章：[说明]`
- 更新设定：`更新《书名》总设定库：[内容]`
- 添加资源：`添加《书名》人物/设定：[名称]`
- 其他更新：`更新《书名》：[说明]`

## 索引化优势

### 传统方式的问题
```
❌ 问题：
- 查找人物需要遍历整个文件夹
- 无法快速了解人物关系
- 难以发现重复或冲突
- 一次性读取所有文件，浪费资源
```

### 索引化方式的优势
```
✅ 优势：
- 通过 list.md 快速定位
- 概况字段一目了然
- 相关资源字段显示关联
- 按需读取，只加载需要的文件
- 便于冲突检测和一致性检查
```

### 实际效果对比

**查找人物"林远"：**

❌ 旧方式：
```python
# 遍历整个文件夹
for f in char_path.glob("*.md"):
    content = read_file(f)
    if "林远" in content:
        ...
```

✅ 新方式：
```python
# 直接搜索 list.md
entries = search_list("人物库", "林远")
# 直接获取文件路径
char_file = char_path / "林远.md"
```

## 最佳实践

1. **添加资源时使用命令**
   - 使用 `/char add`、`/set add`、`/vow add`
   - 自动创建文件并更新 list.md

2. **定期同步检查**
   - 执行 `/list sync 人物库`、`/list sync 设定库`
   - 确保 list.md 与实际文件一致

3. **填写完整信息**
   - 添加资源时填写概况和相关资源
   - 便于后续查找和冲突检测

4. **写作前查阅 list.md**
   - 通过 `/list read 人物库` 了解已有角色
   - 避免重复或冲突

5. **及时更新伏笔状态**
   - 伏笔揭露后执行 `/vow update V001 已揭露`
   - 保持状态准确
