# 📖 写作前检索指南（LLM 主动执行）

**重要：检索由语言模型（LLM）主动提出并执行，不是 Python 脚本。**

---

## 🎯 检索时机

### 必须检索的情况

1. **开始新章节前** - 查阅本章相关资源
2. **涉及重要人物** - 查阅人物档案确保一致性
3. **涉及关键场景** - 查阅场景设定确保连贯
4. **涉及伏笔回收** - 查阅伏笔库确保逻辑闭环
5. **涉及世界观设定** - 查阅设定库确保不矛盾

### 检索触发词

当用户说以下内容时，**必须主动检索**：
- "写第 X 章"
- "继续写"
- "下一章"
- "创作第 X 章"
- "开始写"

---

## 📋 检索流程（LLM 执行）

### 步骤 1：确认本章信息

**向用户确认：**
```
📖 开始创作准备

请确认：
- 章节：第 X 章
- 上一章结尾：（简述悬念）
- 本章核心事件：（用户回答或从大纲推断）
```

### 步骤 2：提出检索计划

**主动告知用户将检索哪些资源：**
```
📚 写作前检索计划

为确保连贯性，我将检索以下资源：

1. **人物库** - 本章出场人物档案
   - 主角名（确认性格、目标）
   - 配角名（确认关系、作用）

2. **设定库** - 本章涉及设定
   - 世界观设定
   - 特殊能力/物品

3. **场景库** - 本章发生地点
   - 地点 A（确认环境描写）

4. **伏笔库** - 相关伏笔状态
   - 伏笔 A（确认是否回收）

5. **前情提要** - 上一章正文
   - 确认结尾悬念

开始检索，请稍等...
```

## 🔍 检索深度控制

### 基础检索（默认）

适用于常规章节：
- 人物库 list.md（前 10 个重要人物）
- 设定库 list.md（前 5 个相关设定）
- 上一章正文
- 本章细纲（如有）

### 深度检索

适用于关键章节（转折章、高潮章、伏笔回收章）：
- 人物库完整档案（所有出场人物）
- 设定库完整设定（所有相关设定）
- 场景库完整场景
- 伏笔库全部未揭露伏笔
- 历史库相关背景
- 前 3 章正文（确保连贯）

### 定向检索

用户明确要求时：
- "查阅林远的档案" → 只读人物库/林远.md
- "看看伏笔列表" → 只读伏笔库/list.md
- "上次设定的修仙体系是什么" → 只读设定库/修仙体系.md

## 📊 检索优先级

### P0（必须检索）

- **本章出场的主要人物档案**（尤其是主角）
- **上一章正文**（确保连贯）
- **本章细纲**（如有）

### P1（强烈建议）

- **相关设定**（世界观、能力体系）
- **场景设定**（地点描写）
- **伏笔状态**（如有涉及）

### P2（按需）

- **历史背景**（如涉及过去事件）
- **配角档案**（如有重要互动）
- **写作要求**（如用户有特殊要求）

---

## ⚠️ 检索注意事项

### 必须检查的一致性

1. **人物一致性**
   - 性格是否符合档案
   - 关系是否正确
   - 当前状态（位置、伤势、情绪）

2. **设定一致性**
   - 世界观规则是否遵守
   - 能力体系是否矛盾
   - 物品设定是否一致

3. **时间线一致性**
   - 上一章结尾时间
   - 本章开始时间
   - 时间跳跃是否合理

4. **伏笔一致性**
   - 已回收的伏笔不再使用
   - 未揭露的伏笔不提前暴露
   - 伏笔回收时机是否合适

### 常见错误及避免

| 错误 | 原因 | 避免方法 |
|------|------|---------|
| 人物性格突变 | 未查阅人物档案 | 写作前必读人物档案 |
| 设定前后矛盾 | 未查阅设定库 | 涉及设定时立即检索 |
| 时间线错乱 | 未读上一章 | 必读上一章结尾 |
| 伏笔遗忘 | 未查阅伏笔库 | 定期检索伏笔状态 |
| 场景描写不一致 | 未查阅场景库 | 涉及地点时检索 |

## 🚀 最佳实践

### DO（应该做的）

- ✅ 每次写作前**主动提出**检索计划
- ✅ 向用户**展示检索报告**
- ✅ **优先检索**人物档案和上一章

- ❌ 检索后不告知用户
- ❌ 忽略人物档案导致性格矛盾

- ❌ 检索过度（常规章节不需要读全部文件）
